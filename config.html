Typing ?dwell or ??G1 X10 F500 opens documentation instead of sending the text to the controller; ?? alone opens this reference unfiltered. A bare ? (status query) is never intercepted. The dialog pauses command streaming while open, so avoid lookups mid-job.
About this reference

Sources. The documentation in this plugin is compiled from:

  • Sienci Labs SLB/SLB‑EXT firmware documentation — the official settings tables, including SLB‑EXT factory defaults and Sienci’s “unused”/“advanced” notes.
  • grblHAL core source — the canonical setting registry and descriptions for mainline (Main_grblHAL) builds, including settings the Sienci tables don’t cover and the current numbering (e.g. laser XY offset at $770/$771, formerly $741/$742 on SLB 5.0.x firmware).
  • grblHAL community docs — the complete G‑code and M‑code reference the command entries are based on.
  • ncSender — the plugin API, and the /api/firmware endpoint (ncSender’s cache of your controller’s own $ES/$$ enumeration) that powers the live “on this controller” check.
  • OctoPrint Marlin G‑code Documentation — the plugin that inspired this one.

Which controllers this applies to. ncSender can drive both grblHAL and FluidNC machines; this reference is written for the grblHAL dialect:

  • Sienci SLB / SLB‑EXT (grblHAL) — fully covered. Defaults are labeled “SLB‑EXT default”, Sienci‑specific settings (button macros $450s, ring/rail LEDs $664/$665, spindle slots $511+) are included, and entries note where Main_grblHAL renumbered older SLB 5.0.x settings.
  • Other grblHAL boards — core G/M‑codes and settings apply as documented; Sienci‑specific entries will simply be flagged “not on this controller” by the live check, since every grblHAL build only exposes the settings its features compile in.
  • FluidNC — the basic G‑code entries (motion, coordinate systems, spindle/coolant M‑codes) largely apply since FluidNC descends from Grbl, but the numbered $ settings do not: FluidNC is configured through a YAML file and $/ tree commands instead. grblHAL extensions (G65 macros, M62–M68 I/O, canned cycles, plugin M‑codes, $ES) are unavailable there, and the live controller check will report “controller settings unavailable”.
  • Classic Grbl 1.1 — the $0–$132 settings and core G/M‑codes align; everything beyond that is grblHAL‑only.

The live check reads ncSender’s cached controller enumeration — after flashing new firmware, refresh ncSender’s Firmware tab once so the cache re‑queries, and this reference picks it up on next open.
